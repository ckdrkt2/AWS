# 클라이언트 - 서버 모델
- 클라이언트는 사람이 컴퓨터 서버에 요청을 보내기 위해 상호 작용하는 웹 브라우저 또는 애플리케이션
- 서버는 요청에 따른 응답을 하기 위해 상호 작용하는 Amazon Elastic Cloud Compute(EC2)와 같은 가상 서버
- 클라이언트가 서버에게 데이터(기사, 그림, 정보 등)을 요청하면 서버는 이 요청을 평가하여 클라이언트에게 요청한 데이터를 반환

 # 클라우드 컴퓨터이 배포 모델
 - 클라우드 전략을 선택하는 기업은 필요한 클라우드 애플리케이션 구성 요소, 선호하는 리소스 관리 도구, 레거시 IT 인프라 요구 사항과 같은 요소를 고려해야 함
 - 배포 모델은 클라우드 기반, 온프레미스, 하이브리드로 구성

 ## 클라우드 기반 배포
 - 애플리케이션의 모든 부분을 클라우드에서 실행
 - 기존 애플리케이션을 클라우드로 마이그레이션
 - 클라우드에서 새 애플리케이션을 설계 및 빌드

 ## 온프레미스 배포(Private 클라우드 배포)
 - 가상화 및 리소스 관리 도구를 사용하여 리소스를 배포
 - 애플리케이션 관리 및 가상화 기술을 사용하여 리소스 활용도를 높임

 ## 하이브리드 배포
 - 클라우드 기반 리소스를 온프레미스 인프라에 결합
 - 클라우드 기반 리소스를 레거시 IT 애플리케이션과 통합
 - 각각의 장점을 필요한 부분에 활용 가능

 # 클라우드 컴퓨팅의 이점
 - 선행 비용을 가변 비용으로 대체
 선행 비용은 미리 투자를 해야 사용할 수 있는 리소스를 사용할 경우에 발생하지만 클라우드 컴퓨팅을 통해 사용한 리소스에 대해서만 비용을 지불하는 가변 비용으로 대체가 가능
 - 데이터 센터 운영 및 유지 관리에 비용 투자 불필요
 - 용량 추정 불필요
 Amazon EC2의 경우 서비스를 시작하고 사용한 리소스에 대해서만 비용을 지불할 수 있고 사용하지 않는 리소스에 대해 비용을 지불하거나 제한된 용량을 사용하는 대신에 필요한 리소스와 용량에 대해서만 비용을 지불하고 사용할 수 있음
 - 규모의 경제로 얻게 되는 이점
 클라우드 컴퓨팅 사용 시 규모의 경제를 통해 인프라를 소유할 때 보다 가변 비용이 낮아짐
 - 속도 및 민첩성 향상
 새로운 리소스나 용량이 필요한 경우에 클라우드의 유연성을 통해서 몇 분 만에 새로운 리소스와 용량을 사용할 수 있음
 - 몇 분 만에 전 세계에 배포
