# Amazon Elastic Compute Cloud(Amazon EC2)
Amazon EC2는 안전하고 크기 조정이 가능한 컴퓨팅 용량을 인스턴스로 클라우드에서 제공
온 프레미스 리소스 사용할 경우
- 하드웨어 구매
- 물리적 데이터 센터에 서버 설치
- 필요한 모든 구성을 수행
Amazon EC2 인스턴스를 사용할 경우
- EC2 인스턴스를 프로비저닝하고 시작할 수 있음
- 워크로드 실행을 완료했다면 인스턴스 사용을 중지할 수 있음
- 사용한 컴퓨팅 시간에 대해서만 비용 지불
- 필요한 서버 용량에 대해서만 비용 지불

# Amazon EC2 작동 방식
1. 시작
    - 기본 구성 인스턴스가 포함되어 있는 템플릿을 선택하여 시작
    - 구성에는 운영 체제, 애플리케이션 서버, 애플리케이션이 포함
    - 인스턴스의 하드웨어 구성인 인스턴스 유형도 선택
    - 인스턴스로 들어오고 나가는 네트워크 트래픽을 제어할 보안 설정을 지정
2. 연결
    - 인스턴스에 연결
3. 사용
    - 명령을 실행하여 소프트웨어 설치, 스토리지 추가, 파일 복사 및 정리 등의 작업 수행

# Amazon EC2 인스턴스 유형
인스턴스 유형은 다양한 작업에 최적화되어 있음. 인스턴스 유형을 선택할 때 컴퓨팅, 메모리, 스토리지 기능에 대한 요구사항을 고려

- 범용 인스턴스
    - 범용 인스턴스는 컴퓨팅, 메모리, 네트워킹 리소스를 균형 있게 제공함.
    - 애플리케이션, 게임, 기업 애플리케이션용 백엔드, 중소 규모 DB 서버로 사용할 수 있음
- 컴퓨팅 최적화 인스턴스
    - 컴퓨팅 최적화 인스턴스는 고성능 프로세서를 활용하는 컴퓨팅 집약적인 애플리케이션에 적합
    - 애플리케이션, 게임 서버와 같은 워크로드에 사용 가능
    - 고성능 웹 서버, 컴퓨팅 집약적 애플리케이션 및 게임 서버에 적합
    - 컴퓨팅 최적화 인스턴스를 단일 그룹에서 많은 트랜잭션을 처리해야 하는 일괄 처리 워크로드에 사용할 수 있음
- 메모리 최적화 인스턴스
    - 메모리 회적화 인스턴스는 메모리에서 대규모 데이터 세트를 처리하는 워크로드를 위한 빠른 성능을 위해 설계
    - 고성능 데이터베이스, 다량의 비정형 데이터의 실시간 처리와 같은 워크로드에서 사용
    - 이외에도 메모리가 많이 필요한 워크로드에서 좋은 성능을 얻을 수 있음
- 액셀러레이티드 컴퓨팅 인스턴스
    - 액셀러레이티드 컴퓨팅 인스턴스는 하드웨어 액셀러레이터 또는 코프로세서를 사용하여 일부 기능을 CPU에서 실행되는 소프트웨어에서보다 더 효율적으로 수행
    - 부동소수점 수 계산, 그래픽 처리, 데이터 패턴 일치 등
    - 데이터 처리를 가속화할 수 있는 구성 요소로
    - 그래픽 애플리케이션, 게임 스트리밍, 애플리케이션 스트리밍과 같은 워크로드에 적합
- 스토리지 최적화 인스턴스
    - 스토리지 최적화 인스턴스는 로컬 스토리지의 대규모 데이터 세트에 대한 순차적 읽기 및 쓰기 액세스가 많이 필요한 워크로드를 위해 설계
    - 분산 파일 시스템, 데이터 웨어하우징 애플리케이션, 고빈도 온라인 트랜잭션 처리(OLTP) 시스템 등에서 사용
    - 지연 시간이 짧은 IOPS(초당 입출력 작업 수)를 애플리케이션이 제공하도록 설계
    - IOPS 요구 사항이 높은 애플리케이션이 있는 경우 스토리지 최적화 인스턴스의 성능이 좋음
